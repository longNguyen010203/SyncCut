# Plans

Use this directory for larger implementation plans.

For non-trivial work, create a plan file before coding.

Example files:
- `docs/plans/build-timeline-mvp.md`
- `docs/plans/remotion-export.md`
- `docs/plans/ffmpeg-assemble.md`

A plan should include:
- goal
- scope
- files to change
- implementation steps
- verification steps
- risks
- out-of-scope items
