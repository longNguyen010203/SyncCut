## Context and Orientation

The repository root is `/home/longnguyen/Desktop/AI/Codex/SyncCut`. SyncCut contains a Python CLI package under `synccut/` and a standalone Remotion project under `remotion/`.

The Python CLI uses Typer. Typer is a Python library that turns annotated Python functions into command-line commands and help output. The public CLI entrypoint is the console script `synccut`, configured in `pyproject.toml`.

The Remotion project is a Node/npm package that renders React components as video. It lives under `remotion/` and consumes `remotion/props.json`. Remotion rendering is intentionally driven by npm scripts, not by Python commands.

The release checklist source for this phase is `docs/release-checklist-v0.1.md`. It is procedural and command-oriented. This hardening phase should run that checklist, record evidence, and produce a clear readiness summary.

The current MVP capability is:

- `synccut build-timeline` reads prepared `scenes.json`, section audio files, and alignment JSON files and writes `timeline.json`.
- `synccut validate-timeline` validates timeline structure and timing.
- `synccut inspect` prints a human-readable section and scene overview.
- `synccut export-remotion` writes `remotion/props.json`.
- `synccut prepare-remotion-assets` copies section audio into `remotion/public/audio/` and annotates props.
- `synccut prepare-visual-assets` optionally copies local `AI_VIDEO` and `B_ROLL` visual files into `remotion/public/visuals/`.
- `synccut inspect-visual-assets` reports optional visual readiness.
- `synccut preflight` reports render readiness from props and can verify public files with `--verify-files --public-dir`.
- `npm run typecheck` validates the Remotion TypeScript project.
- `npm run still:local`, `npm run render:smoke:local`, `npm run render:segment:local`, and `npm run render:final:local` are local Remotion workflows that use `/usr/bin/google-chrome`.

v0.1 means this prepared-input MVP is ready as a baseline for users who can provide scene planning JSON, narration audio, and alignment JSON. It does not mean SyncCut parses DOCX, generates AI media, downloads B-roll, calls ffmpeg directly, probes or transcodes media, offers a GUI, or invokes Remotion rendering from Python.
